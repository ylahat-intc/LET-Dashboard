Power Automate Flows

Flow 1: LET - BU Rep Approval v2 (Automatic) ✓

Trigger: When item is created  |  Purpose: Auto-start BU rep approval on new request

✓ Rebuilt clean June 2026 with correct Filter array catchall logic.

Structure:

  1. Trigger (item created on main LET list)
  2. Get item → main LET list → ID from triggerBody()?['ID']
  3. Get items → BU Reps list → filter: field_1 eq '[Get item field_7]'
  4. Filter array → finds catchall rep: and(empty(item()?['NextLevelScope']),equals(item()?['field_1'],outputs('Get_item')?['body/field_7']))
  5. Condition: length(body('Filter_array')) > 0
    • TRUE (catchall exists): Send approval to first(body('Filter_array'))?['RepEmails'] → Update item on response
    • FALSE (no catchall): Apply to each all reps → Send approval to each → Update item on response

Update item fields on approval: BURepApprovalStatus, BURepApproverEmail, BURepApprovalDate (utcNow('yyyy-MM-dd')), WorkflowStage

Flow 2: LET - Manual BU Rep Approval v2 ✓

Trigger: For a selected item (V2)  |  Purpose: Re-trigger approval for existing items

✓ Working as of June 2026.

How to use: SharePoint list → check item checkbox → Flows button → select flow → Run

Same logic as Flow 1 but uses triggerBody()?['ID'] to read the selected item.

Flow 3: LET - Net Growth Request Notification ✓

Trigger: When item is created or modified  |  Purpose: Notify RE Director of Net Growth requests (once only)

  • Condition: LETClassification/Value = "Net Growth Request" AND NetGrowthEmailSent = false
  • To: adam.novak@intel.com  |  CC: yariv.lahat@intel.com
  • Subject: ⚠️ Net Growth Request - LET #[Title] requires your review
  • After sending: sets NetGrowthEmailSent = true to prevent repeat emails
⚠ Circular loop warning on save is expected and harmless — the flag blocks re-triggering.

Flow 4: Change Notification Email (existing) ✓

Flow ID: 93389d4c-011d-43c6-b806-2c7db572e3a6  |  Trigger: When item is created or modified

Updated to include Approval Information section:

Common Gotchas & Fixes

IssueFix
